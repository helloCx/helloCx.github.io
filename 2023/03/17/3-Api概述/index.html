普通 Api

http://localhost:8080/api/get_list?id=1

http://localhost:8080/api/post_list?id=1

直接通过接口接口名实现

RestFul Api

http://localhost:8080/api/list/1一个接口完成所有查询、参数直接带在斜杠后面

特点：通过请求类型区分POST 还是GET/ 路径都是同一个

实现：通过动态路由实现

版本控制

nest支持四种类型

类型 传递方式
URI 版本管理 版本在请求的 URI 中传递 （默认）
Header 版本管理 在自定义的 Header 中传递版本
Media Type 版本管理 在 Accept 头部标签中声明版本
自定义版本管理 请求的任何部分都可用于指定版本，需要提供一个自定义函数来提取所述版本
